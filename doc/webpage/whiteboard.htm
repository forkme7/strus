Introspection of Strus objects on client side
(LONG-TERM)
Something missing in strus is introspection of the data structures built in a program (expression trees, function definitions, data loaded and processed etc.).
Because this seems to be an aspect on its own, we could implement an own proxy universe from StorageObjectBuilderInterface and AnalyzerObjectBuilderInterface, similarly to what strusModule (the module loader) or strusRpc does. Every object created would implement its method calls as redirect to the real method call and additionaly build some representation in a universe that is introspectable (an object tree).
