Strus installation guide

Releasing

There are no tagged releases yet. You build from trunk. There will be a first release version 0.1. soon. The development status is pre-alpha.

Fetch it from github

You have to fetch the projects from github and build them on your own. There are no packages yet. In order to fetch them from github by cloning it with: 

git clone git@github.com:patrickfrey/<projectname>.git
or by downloading and unpacking it with: 

wget https://github.com/patrickfrey/<projectname>/archive/master.zip
unzip master.zip
The following projects (substitute for <projectname>) you need to fetch: strus, strusAnalyzer, strusModule, strusRpc, strusBindings, strusUtilities.

Plattforms supported

Currently only my development platform Ubuntu 14.04.2 LTS and the deployment platform for the Wikipedia search 4.0.5-1-ARCH is guaranteed to be built without problems. On other platforms with a different gcc, you might get compile warnings that are treated as errors. Please contact me immediately if you run into problems. Build errors on any Linux I will handle with top priority.

3rd party dependencies

Hier is a list of 3rd party software you need to install for strus:

  • boost (>= 1.53)
  • leveldb (needed by strus)
  • libevent (2.x) (needed by strusRpc)
  • SWIG (>= 2) (needed by strusBindings)
  • gperftools(needed by leveldb)
  • libunwind (needed by leveldb)
  • snappy (needed by leveldb)

Build

The build system is CMake. Go into the top level directory of each project and execute the following: 

cmake CMakeLists.txt
make
sudo make install
The projects will by default be installed in the default location for utilities (On Ubuntu in /usr/local). The projects have due to dependencies be build and installed in the following order (projects on the same line can be build in reversed order too because they have no dependencies to each other):
  1. strus, strusAnalyze
  2. strusModule
  3. strusRpc
  4. strusBindings, strusUtilities
