Strus

With strus you can build full-text search engines with elementary bricks.

  • Status
    Development status is "Beta".
  • License

  • Download

About

The project strus provides a collection of C++ libraries for building a full-text search engine. The strus search engine can be build using any key value store database that provides an upper bound seek function for the stored key/value pairs. Currently there exists an implementation based on the google leveldb library.

Components

Strus consists of several components represented as sub project on github:

  1. strus

    The storage and query processing of the search engine

  2. strusAnalyzer

    The document and query analysis for transforming content into retrievable items and searchable terms.

  3. strusModule

    Provides the loading of search engine components from dynamically loadable modules. (depends on strus and strusAnalyzer)

  4. strusUtilities

    Provides some command line programs to access the search engine. (Depends on strusModule)

  5. strusBindings

    Provides language bindings to use strus with other programming languages like PHP,Python,Perl,Java,Ruby,etc... (Depends on strusModule)

Some components to implement a competitive search are still missing. For example the possibility to distribute the search index and running strus as server nodes in a cluster without too much configuration and administration effort. Strus has interfaces to populate and receive statistics that have to be shared in order to make ranking weights of different search engine nodes comparable. But strus as server with the messaging to exchange this data and the orchestration of server nodes is not implemented yet.
