Strus documentation

Interface documentation

Strus provides two classes of interfaces with diametric objectives. One is the C++ interface that is the base of the implementation and the other is the interface for language bindings with implementations that wrap to the C++ interface.

C++ interfaces

  1. strus core: Interface to storage and query evaluation
  2. strus analyzer: Interface to process documents and queries in strus.
  3. strus module loader: Interface to define strus components and to load them from dynamically loadable modules.
  4. strus RPC client: Interfaces for strus RPC client and server
  5. strus program loader: Functions to instantiate strus components from source

Language bindings

The language binding interfaces are for scripting languages calling strus functions. Language bindings can call strus functions directly or they can call functions on objects residing on a server via RPC. Command line tools documentation

You do not need the command line tools of strus. All functionality is accessible with the API.
But there exist a lot of command line tools helpful for access and maintenance of a strus storage. A list of the standard command line tools an their documentation can be found here (utilities).
